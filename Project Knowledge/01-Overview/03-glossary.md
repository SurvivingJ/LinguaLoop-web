# LinguaLoop Glossary

> **Canonical Terminology Reference** — This glossary defines the authoritative meaning of terms used throughout the LinguaLoop project. When documentation or code comments refer to these terms, the definitions below apply.

---

## Domain Terms

| Term | Definition |
|------|------------|
| **CEFR** | Common European Framework of Reference for Languages. The platform's difficulty levels 1–9 map to CEFR codes (A1, A1+, A2, A2+, B1, B1+, B2, C1, C2). |
| **ELO** | Rating system adapted from chess, used for adaptive test matching. Users start at 1200; tests start at varying ELO based on their difficulty level. |
| **OTP** | One-Time Password. Used for passwordless authentication via email. |
| **Slug** | URL-friendly unique identifier for a test (e.g., `the-history-of-tea-cn-3`). |
| **Transcript** | The reading or listening passage text that forms the basis of a test. |
| **Prose** | Synonym for transcript, used within the test generation pipeline context. |
| **Test** | A listening, reading, or dictation comprehension exercise (NOT a unit test). Contains a transcript and 5 questions. |
| **Question** | A multiple-choice item within a test, with 4 choices and 1 correct answer. |
| **Test Attempt** | A single instance of a user taking a test, recording the score and ELO changes. |
| **Difficulty Level** | Integer 1–9 mapping to the CEFR framework. |
| **Skill Type / Test Type** | The mode of test interaction: listening, reading, or dictation. |
| **Language** | Target language being learned (Chinese, English, Japanese). Identified by ID (`1`=cn, `2`=en, `3`=jp) or language code. |
| **Token** | In-app currency used to take tests (1 token) or generate tests (5 tokens). Users receive 2 free tokens daily. |
| **Lens** | A perspective or angle applied during topic generation (e.g., historical, cultural, economic, scientific). |
| **Topic** | A subject for test content, generated by the topic generation pipeline. |
| **Category** | A broad grouping of topics (e.g., "Technology", "Nature", "Sports"). |
| **Production Queue** | Staging area that links approved topics to languages for test generation. |
| **Dimension Table** | Reference/lookup tables prefixed with `dim_` (e.g., languages, test types, question types, lens, status). |
| **Semantic Signature** | A text string combining category, concept, lens, and keywords, used for embedding-based deduplication. |
| **Embedding** | A 1536-dimensional vector representation of text, used for similarity search via pgvector. |
| **Volatility** | A measure of ELO rating uncertainty; decreases as a user completes more attempts. |

## Internal Naming Conventions

| Term | Definition |
|------|------------|
| **LinguaLoop vs LinguaDojo** | The project uses both names. LinguaLoop is the project name; LinguaDojo appears in code and public-facing URLs (e.g., `audio.linguadojo.com`). |
| **anon client vs service client** | Two Supabase clients. The anon client respects Row-Level Security (RLS); the service client bypasses RLS for admin operations. |
| **gen_user** | The user ID credited as creator of an AI-generated test. |
| **system_user_id** | A hardcoded UUID for system-generated content (`de6fd05b-0871-45d4-a2d8-0195fdf5355e`). |
| **tier** | User subscription level: free, premium, admin, or moderator. |
| **test_mode** | Frontend term for test type (listening, reading, or dictation). |
| **Blueprint** | Flask URL grouping mechanism (e.g., `auth_bp`, `tests_bp`, `reports_bp`). |

## Abbreviations

| Term | Definition |
|------|------------|
| **RLS** | Row-Level Security — a Supabase/PostgreSQL feature that restricts row access based on policies. |
| **JWT** | JSON Web Token — used for authentication. |
| **TTS** | Text-to-Speech — powered by Azure Cognitive Services. |
| **RPC** | Remote Procedure Call — Supabase database functions invoked from the client. |
| **R2** | Cloudflare R2 — S3-compatible object storage used for audio and other assets. |
